# CryoCore
> From Origin to Destination. Every Degree Matters.

## What is CryoCore

CryoCore is a multitenant cold-chain logistics platform built for the pharmaceutical industry. It has two dedicated portals — a Client Portal for pharma companies to place and track orders, and an OPS Control Tower for logistics teams to manage shipments, routes, inventory, compliance, and alerts — all from a single unified system.

Pharmaceutical products like vaccines, biologics, insulin, and monoclonal antibodies require strict temperature-controlled environments throughout their supply chain. A single temperature excursion can render an entire shipment unusable, costing millions and endangering patient lives.

CryoCore solves this by giving both pharma clients and logistics teams real-time visibility, intelligent route planning, and unified compliance tracking in one platform.

### Target Users
- **Pharmaceutical companies** who ship temperature-sensitive products globally.
- **Logistics operations teams** who manage carriers, routes, and compliance.

## Key Features

### Client Portal
- Place and track pharmaceutical orders
- Real-time temperature status per shipment
- Order history with shipment pipeline progress
- Analytics dashboard with cost and risk data

### OPS Control Tower
- Live global shipment map with animated routes
- Intelligent route planner with Air/Sea/Rail/Road comparison by cost, speed, and risk score
- Inventory management for cold-chain materials
- Compliance document approval and audit logs
- Predictive risk analytics with AI recommendations
- Alert management with critical/warning/resolved tabs

## System Architecture

### High Level Flow
Client places order → OPS receives notification → OPS opens Route Planner → Selects optimal route → Shipment assigned to carrier → Real-time tracking begins → Temperature monitored throughout transit → Compliance verified → Delivered to destination

### Architecture Description
CryoCore is a frontend-only proof of concept built with pure HTML, CSS, and JavaScript. All data is simulated through mock data arrays. The platform demonstrates a complete multitenant SPA (Single Page Application) architecture with two separate portals sharing a common design system.

### Production Architecture (Planned)
- **Frontend:** React with TypeScript
- **Backend:** Node.js with Express
- **Real-time:** WebSockets via Socket.io
- **IoT:** MQTT broker with AWS IoT Core
- **Message Queue:** Apache Kafka for sensor data
- **Deployment:** AWS with Docker and Kubernetes

## Database Design

*Note: This is a frontend POC — no actual database is implemented. The following describes the intended data model for production.*

### Entities
- **Users**: Client tenants and OPS users
- **Orders**: Placed by clients, assigned to shipments
- **Shipments**: Linked to orders, carriers, routes
- **Inventory**: Cold-chain materials and stock levels
- **Compliance Documents**: Linked to shipments
- **Alerts**: Generated by shipments and inventory
- **Routes**: Air/Sea/Rail/Road options per shipment
- **Carriers**: DHL, FedEx, UPS, Kuehne+Nagel, Marken

## End to End Workflow

1. Pharma client visits CryoCore landing page.
2. Client logs in via Client Portal button.
3. Client places a new pharmaceutical order with product, quantity, and temperature zone.
4. OPS team receives notification in bell icon.
5. OPS clicks notification which opens the Route Planner with order details pre-filled.
6. OPS compares Air/Sea/Rail/Road route options by cost, transit time, and risk score.
7. OPS selects optimal route and assigns it to a carrier.
8. Shipment appears in live global map with animated route line and status dot.
9. Client tracks shipment progress through: Ordered → Shipped → In Transit → Distribution → Delivered.
10. OPS monitors temperature compliance, resolves alerts, and approves documents.
11. Shipment delivered, compliance logged.
